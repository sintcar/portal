# Пошаговые действия после переноса репозитория из GitHub в пустой каталог

Ниже — подробный чек-лист, который поможет убедиться, что проект корректно перенесён, окружение настроено, а приложение запускается локально.

## 1) Проверить целостность копии
- Убедитесь, что находитесь в корне проекта: `pwd`.
- Посмотрите содержимое верхнего уровня, чтобы убедиться, что нужные файлы на месте (README, манифесты пакетов и т. д.): `ls`.
- Если внутри уже есть история Git (например, при `git clone --depth=1`), убедитесь, что рабочее дерево чистое: `git status`.

## 2) Прочитать документацию проекта
- Откройте `README.md` и дополнительные файлы (`docs/`, `TECHNICAL_DOCUMENTATION.md`, модульные README), чтобы узнать о требованиях и командах.
- Обратите внимание на версии зависимостей (Node, Python, Java и т. д.) и платформенные особенности.

## 3) Установить необходимые инструменты
- Поставьте рантаймы и менеджеры пакетов, указанные в документации (например, Node + npm/pnpm/yarn; Python + pip/pipenv/poetry; Go; Java + нужный билд-инструмент).
- Убедитесь, что вспомогательные сервисы доступны (PostgreSQL, Redis, RabbitMQ и т. д.), если они нужны.

## 4) Настроить переменные окружения
- Найдите шаблон `.env.example` (или аналог) и скопируйте его в `.env`: `cp .env.example .env`.
- Заполните все требуемые значения (ключи, URL сервисов). Если есть несколько сервисов (frontend/backend/worker), настройте каждую их окружения отдельно.

## 5) Установить зависимости
- Backend-примеры: `pip install -r requirements.txt` или `poetry install`, `npm install`, `bundle install` — в зависимости от стека.
- Frontend-примеры: `npm install` / `pnpm install` / `yarn install`.
- В монорепозитории проверьте, нужно ли устанавливать зависимости в отдельных пакетах или через workspace.

## 6) Подготовить базы данных и ассеты
- Выполните миграции: например, `alembic upgrade head`, `python manage.py migrate`, `rails db:migrate`, `prisma migrate deploy` — согласно стеку.
- При необходимости загрузите сиды: `npm run seed`, `python manage.py loaddata`, `rails db:seed` и т. д.
- Соберите статические ассеты, если это требуется: `npm run build` (или команда из документации).

## 7) Прогнать линтеры и тесты
- Запустите линтеры и тесты, указанные в проекте: `npm run lint`, `npm test`, `pytest`, `go test ./...`, `mvn test` и т. д.
- Если появляются ошибки окружения (нет переменных или сервисов), устраните их перед дальнейшей работой.

## 8) Запустить приложение локально
- Используйте команды из документации: `npm start`, `npm run dev`, `python manage.py runserver`, `rails s`, `docker compose up` и т. д.
- Для нескольких сервисов (API + frontend + воркеры) запустите каждый, либо используйте `docker compose` при наличии конфигурации.

## 9) Проверить работоспособность
- Откройте указанный локальный адрес (часто `http://localhost:<port>`) и проведите быстрые проверки:
  - Загружается ли главная страница или health-endpoint API?
  - Получается ли войти или выполнить пример операции (если даны тестовые учётные данные)?
- Следите за логами сервисов при навигации, чтобы заметить ошибки.

## 10) Инициализировать или обновить Git (при необходимости)
- Если `.git` отсутствует, инициализируйте репозиторий: `git init`, затем добавьте файлы `git add .` и сделайте первый коммит, когда всё готово.
- Если Git уже есть, держите рабочее дерево чистым и добавьте нужные записи в `.gitignore` перед коммитами.
- Настройте имя и email: `git config user.name "Ваше Имя"` и `git config user.email you@example.com`.

## 11) Настроить инструменты качества
- Если рекомендуется, установите pre-commit хуки: `pre-commit install`.
- Настройте редактор согласно форматам проекта (`.editorconfig`, ESLint/Prettier/Black/ruff и др.).

## 12) Подготовка к командной работе
- Если будете пушить в удалённый репозиторий:
  - Добавьте remote: `git remote add origin <repo-url>`.
  - Синхронизируйтесь с основной веткой: `git pull origin main` (или другой default branch).
- Для соответствия CI изучите конфигурации (`.github/workflows/`, `.gitlab-ci.yml`, `circleci/`) и повторите требуемые команды локально.

Следуя этому чек-листу, вы проверите перенос, установите зависимости, подготовите окружение и запустите сервисы для дальнейшей разработки или отладки.
